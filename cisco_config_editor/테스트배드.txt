완성된 Cisco Config Manager의 모든 기능을 철저하게 검증하기 위한 통합 테스트 시나리오를 제안합니다.

장비가 **IOS-XE (Catalyst 계열)**와 NX-OS (Nexus 계열) 두 대가 있으므로, 이 환경에 맞춰 호환성과 기능 동작을 모두 확인할 수 있도록 구성했습니다.

🧪 테스트 전 준비 사항
PC 환경: Python 가상환경 활성화 및 필요 라이브러리 설치 (netmiko, paramiko, matplotlib, networkx).

네트워크: PC에서 두 스위치(IOS-XE, NX-OS)로 Ping이 나가는지 확인.

계정: 두 장비에 SSH 접속이 가능한 계정 정보 준비 (Username/Password/Enable Password).

📍 시나리오 1: 장비 등록 및 연결 (Connection Manager)
목표: DeviceManager와 ConnectionWorker가 실제 장비와 통신하는지 확인.

장비 등록:

상단 툴바의 장비 관리 버튼 클릭.

장비 추가 클릭.

IOS-XE 장비: IP, 계정 입력, 타입 cisco_iosxe 선택.

NX-OS 장비: IP, 계정 입력, 타입 cisco_nxos 선택.

확인: 리스트에 두 장비가 추가되는지 확인.

연결 테스트:

두 장비의 체크박스를 선택하고 🔌 연결 버튼 클릭.

확인: 상태가 연결됨으로 바뀌고 초록색으로 표시되는지 확인.

확인: 로그 탭에 "Connected to..." 메시지가 찍히는지 확인.

터미널 테스트:

터미널 탭으로 이동.

IOS-XE 장비 선택 후 show ver 입력 및 실행 -> 결과 출력 확인.

NX-OS 장비 선택 후 show ver 입력 및 실행 -> 결과 출력 확인.

📍 시나리오 2: 설정 가져오기 및 파싱 (Pull & Parse)
목표: CLIAnalyzer가 각 OS의 설정을 제대로 파싱하여 UI에 뿌려주는지 확인.

IOS-XE 장비 Pull:

메인 창 상단 콤보박스에서 IOS-XE 장비 선택.

📥 현재 설정 가져오기 (Pull) 클릭.

비밀번호 입력 후 대기.

데이터 검증 (UI 확인):

구성 트리 (좌측): 호스트명, 인터페이스 목록, VLAN 목록이 트리 형태로 나타나는지 확인.

글로벌 탭: 호스트명이 장비 실제 호스트명과 일치하는지 확인.

인터페이스 탭: 인터페이스 목록에 실제 포트들(Gi1/0/1 등)이 들어왔는지 확인. 하나를 클릭했을 때 IP 주소나 Description이 제대로 로드되는지 확인.

VLAN 탭: 장비에 설정된 VLAN 리스트가 테이블에 채워졌는지 확인.

📍 시나리오 3: 데이터 모델링 & Undo/Redo 검증 (핵심 수정 사항)
목표: 탭 이동 시 데이터 유지 여부와 Ctrl+Z가 리스트 항목까지 제어하는지 확인.

데이터 유지 테스트:

인터페이스 탭에서 추가 버튼 클릭 -> 이름: Loopback999 입력 -> 확인.

생성된 Loopback999 선택 -> IP Address에 10.10.10.1 255.255.255.255 입력 -> Description에 Test Interface 입력.

다른 탭(예: VLAN 탭)으로 이동.

다시 인터페이스 탭으로 복귀 -> Loopback999 선택.

확인: 입력했던 IP와 Description이 그대로 남아있어야 함.

Undo/Redo 테스트:

위 상태에서 키보드 Ctrl+Z를 한 번 누름.

확인: Description이 지워지거나 IP가 지워지는지 확인.

계속 Ctrl+Z를 누름.

확인: 결국 리스트에서 Loopback999 자체가 사라져야 함 (ListRemoveCommand 동작 확인).

Ctrl+Y (Redo)를 눌러서 다시 나타나는지 확인.

📍 시나리오 4: 설정 변경 및 배포 - IOS-XE (Push)
목표: CiscoCommandGenerator가 생성한 명령어가 실제 장비에 적용되는지 확인.

변경 사항 작성:

VLAN 탭: VLAN 추가 -> ID: 50, 이름: Test_VLAN_50 추가.

인터페이스 탭: 임의의 포트(예: Gi1/0/24) 선택 -> Shutdown 체크 해제, Description에 Deployed by App 입력, 모드 L2 Access, Access VLAN 50 입력.

명령어 생성:

도구 -> 명령어 생성 (F5) 클릭.

우측 생성될 명령어 (Preview) 패널 확인.

확인: vlan 50, name Test_VLAN_50, interface GigabitEthernet1/0/24, switchport access vlan 50 등의 명령어가 정확히 생성되었는지 확인.

배포 (Push):

🚀 설정 장비에 적용 (Push) 클릭.

배포 시작 클릭.

확인: 배포 결과창에 성공 메시지가 뜨는지 확인.

장비 확인:

실제 장비 콘솔이나 장비 관리 -> 터미널에서 show vlan id 50, show run int gi1/0/24로 설정이 들어갔는지 확인.

📍 시나리오 5: NX-OS 호환성 테스트
목표: NX-OS 장비에서도 기본 기능이 작동하는지 확인 (명령어 차이 주의).

Pull 테스트:

메인 창에서 NX-OS 장비 선택 -> Pull 수행.

확인: 인터페이스 명칭(Ethernet1/1 등)이 정상적으로 파싱되는지 확인.

Push 테스트 (주의):

NX-OS는 write memory 대신 copy run start를 씁니다. 앱의 CiscoCommandGenerator는 기본적으로 write memory를 생성합니다.

테스트: VLAN 탭에서 VLAN 60 추가.

명령어 생성 확인.

Push 버튼 클릭 -> 다이얼로그의 명령어 편집창에서 맨 마지막 줄 write memory를 copy run start로 수동 수정(혹은 삭제) 후 배포.

확인: NX-OS에 VLAN 60이 생성되었는지 확인.

📍 시나리오 6: 고급 기능 (템플릿 & 토폴로지)
목표: 부가 기능들의 정상 동작 확인.

템플릿 관리:

현재 IOS-XE 장비의 설정이 로드된 상태에서 템플릿 -> 현재 구성을 템플릿으로 저장 클릭.

이름: My_Switch_Template 저장.

파일 -> 새 구성 클릭 (화면 초기화).

템플릿 -> 템플릿 관리자 열기 -> 방금 저장한 템플릿 선택 -> 불러오기.

확인: 초기화되었던 화면에 이전 설정 데이터가 다시 로드되는지 확인.

토폴로지 맵:

도구 -> 네트워크 토폴로지 (F10) 클릭.

장비 추가: 이름 IOS-XE, 타입 L3 Switch, IP 입력 -> 추가.

장비 추가: 이름 NX-OS, 타입 L3 Switch, IP 입력 -> 추가.

링크 추가: 출발지 IOS-XE, 도착지 NX-OS 선택 -> 추가.

확인: 화면 중앙에 두 장비가 연결된 그림이 그려지는지 확인. (Matplotlib 필요)

⚠️ 테스트 중 발생 가능한 이슈 및 대처
SSH 연결 실패: 방화벽, ACL, 혹은 장비의 ip ssh version 2 설정 여부를 확인하세요.

명령어 에러 (NX-OS): 앱이 생성하는 명령어는 기본적으로 IOS 표준입니다. NX-OS에서 문법이 다른 경우(예: write memory), 배포 직전 다이얼로그의 텍스트 에디터에서 명령어를 수정해서 보내시면 됩니다.

권한 문제: 장비 접속 계정이 privilege 15 권한을 가지고 있는지 확인하세요.

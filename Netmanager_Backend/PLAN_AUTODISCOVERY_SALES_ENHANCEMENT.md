# 오토디스커버리 및 자동 토폴로지 판매 강화 종합 계획

## 1. 기술적 성숙도 평가 및 개선 영역

### 현재 상태
*   **탐지 엔진 (Discovery Engine):** 표준 SNMP (v2c), Ping/Nmap (CIDR), 재귀적 이웃 탐색 (LLDP/CDP)에 의존.
    *   *성숙도:* **레벨 3 (기능적)**. 표준 엔터프라이즈 LAN에서는 작동하지만, 심층적인 WAN/SD-WAN 가시성이나 복잡한 멀티벤더 식별 능력은 부족함.
*   **토폴로지 시각화:** ReactFlow 기반이며 기본적인 그룹화(사이트) 및 레이어 토글 기능 제공.
    *   *성숙도:* **레벨 3 (상호작용적)**. 정적 분석에는 좋으나, 대규모 네트워크(>1000 노드)를 위한 고급 자동 배치 알고리즘과 "타임 머신"(과거 시점 조회) 기능이 부족함.
*   **L3/라우팅:** OSPF에 대한 기본 드라이버 지원은 있으나 활용도가 낮음.
    *   *성숙도:* **레벨 2 (기초적)**. NetBrain이나 Auvik과 경쟁하려면 전체 BGP/MPLS/VXLAN 가시화가 필요함.

### 핵심 개선 필요 영역
1.  **장비 지문 인식 (Device Fingerprinting):** `identify_device` 로직 강화. 단순 `sysDescr` 파싱을 넘어, 포괄적인 OID 서명 데이터베이스를 구축하여 "Unknown Vendor" 결과를 제거해야 함.
2.  **이벤트 기반 토폴로지 (Event-Driven Topology):** 단순 "예약 스캔"에서 "이벤트 트리거 업데이트"로 전환.
    *   *실행 방안:* `Syslog/SNMP Trap` 리스너를 `realtime_event_bus`에 연결하여 `LINK_UP`/`LINK_DOWN` 발생 시 즉시 마이크로 스캔 트리거.
3.  **L3 및 오버레이 시각화:** OSPF/BGP 이웃 매핑 완성. VLAN 스패닝 트리 및 VXLAN 터널에 대한 논리적 뷰 추가.

## 2. 핵심 가치 제안 및 차별화 포인트

### 핵심 가치 제안 (Value Proposition)
> "수동으로 네트워크 지도를 그리는 것은 그만두세요. NetManager는 케이블이 연결되는 순간 즉시 업데이트되는, 살아 숨 쉬는 인프라 지도를 제공합니다."

### 주요 차별화 요소
1.  **"Zero-Touch" 심층 분석:** 복잡한 에이전트 설치가 필요한 경쟁사와 달리, 에이전트리스 방식(SNMP/SSH)으로 심층적인 재귀 크롤링을 수행.
2.  **하이브리드 토폴로지 (물리 + 논리):** 단순한 케이블 연결뿐만 아니라 *트래픽 경로* (라우팅 + 스위칭 로직 결합)를 시각화.
3.  **가성비:** 레거시 NMS (SolarWinds, NetBrain) 대비 훨씬 합리적인 비용으로 엔터프라이즈급 매핑 제공.

## 3. 기술적 한계점 분석 및 고급 기능 추가 방안

### 현재 한계점
*   **폴링 지연:** 네트워크 변경 사항 반영에 시간이 걸림 (스케줄러 주기에 의존).
*   **규모:** 5,000 노드 이상일 경우 `ReactFlow` 성능 저하 가능성 (WebWorkers/Canvas 최적화 필요).
*   **사각지대:** 방화벽이나 비표준 SNMP 장비(IoT)가 보이지 않거나 일반 장비로 표시됨.

### 고급 기능 추가 (Advanced Features)
1.  **실시간 네트워크 매핑:**
    *   *기술:* Trap/Syslog 수신 즉시 WebSocket을 통해 토폴로지 업데이트 푸시.
2.  **AI 기반 장비 식별:**
    *   *기술:* 머신러닝을 활용해 오픈 포트 패턴(Nmap 지문)과 MAC OUI를 분석하여 "Unknown" 장비를 자동 분류.
3.  **예측 토폴로지 (What-If 분석):**
    *   *기술:* 토폴로지 그래프에서 링크 장애를 시뮬레이션하여 라우팅 경로 변경을 *사전에* 예측.
4.  **클라우드/하이브리드 통합:**
    *   *기술:* AWS VPC Flow Logs / Azure VNet API 어댑터를 통해 물리적 스위치와 클라우드 노드를 함께 표시.

## 5. 시장 적합성 검증 및 우선순위

### 타겟 세그먼트
1.  **MSP (Managed Service Providers):** 최우선 순위. 다수 고객사의 네트워크를 관리하며 자동 문서화가 절실함.
2.  **중견 기업 (제조/유통):** 가시성은 필요하지만 1억 원대 NMS를 도입하기 어려운 기업.

### 검증 전략
*   **설문조사:** 기존 사용자 5명 인터뷰. "현재 네트워크 지도의 가장 불편한 점은 무엇인가?"
*   **베타 테스트:** "Auto-Discovery 2.0" 버전을 3곳의 MSP 고객에게 비공개 배포.

## 6. 경쟁 우위 확보 및 특허 전략

### 경쟁사 분석
*   **NetBrain:** 고비용, 무거운 설정. -> *우리의 우위: 경량화, 최신 UI, 합리적 비용.*
*   **Auvik:** 클라우드 전용. -> *우리의 우위: 보안 민감 고객을 위한 온프레미스 옵션 제공.*
*   **SolarWinds:** 구식 UI, 복잡함. -> *우리의 우위: UX 중심 설계.*

### IP (지식재산권) 전략
*   **특허 가능성:** "재귀적 다계층 토폴로지 상관관계 알고리즘" (L2 LLDP와 L3 OSPF, ARP 테이블을 단일 통합 그래프로 병합하는 기술).
*   **방어 전략:** 복제하기 어려운 벤더별 OID 동작 데이터베이스 구축.

## 7. 단계별 개발 로드맵

### 1단계: PoC (개념 증명) - 1개월 차
*   **목표:** "Unknown" 벤더 문제 해결 및 실시간 업데이트 기반 마련.
*   **작업:**
    *   범용 OID 스캐너 구현.
    *   Syslog 리스너를 토폴로지 새로고침(Refresh)과 연동.
*   **성공 지표:** 장비 인식률 > 90%.

### 2단계: MVP (최소 기능 제품) - 2~3개월 차
*   **목표:** 판매 가능한 수준의 UI 및 경로 추적(Path Trace) 완성.
*   **작업:**
    *   UI 고도화 (아이콘, 애니메이션).
    *   경로 추적 시각화 (출발지 -> 목적지).
    *   지도 PDF/Visio 내보내기 기능.
*   **성공 지표:** 비기술자 이해관계자에게 데모 시연 시 긍정적 반응 확보.

### 3단계: 베타 (Beta) - 4개월 차
*   **목표:** 규모 확장 및 안정성.
*   **작업:**
    *   부하 테스트 (1,000+ 노드).
    *   3개 고객사 대상 파일럿 운영.
*   **성공 지표:** 1주일 연속 운영 시 크래시 0건.

### 4단계: 정식 출시 (Launch) - 5개월 차
*   **목표:** 수익 창출.
*   **작업:**
    *   마케팅 캠페인 시작.
    *   공식 문서 배포.

## 10. 사업 계획 및 재무 분석

### 예상 투자 비용 (6개월)
*   **개발비:** 시니어 백엔드 2명, 프론트엔드 1명 (약 1.5억 원 환산 가치).
*   **마케팅/영업:** 3천만 원 (광고, 콘텐츠 제작).
*   **인프라:** 5백만 원 (테스트 랩, 클라우드).
*   **총 소요 비용:** 약 1.85억 원.

### 수익 예측
*   **가격 모델:** 장비당 구독형 (연간 $5/장비) 또는 노드 단위 영구 라이선스.
*   **손익분기점:**
    *   100노드 팩 영구 라이선스를 $5,000에 판매 시 -> 약 37건 판매 시 손익분기 도달.
    *   목표: 1차년도 고객 50사 확보.

### 위험 요인 및 대응
*   **위험:** 고객사 내부망의 SNMP 보안 제약.
    *   *대응:* 보안 터널링을 지원하는 경량 "위성 에이전트(Satellite Agent)" 개발.
*   **위험:** 토폴로지 무한 루프 (스위칭 루프).
    *   *대응:* 그래프 알고리즘 내 루프 감지 및 시각적 경고 로직 구현.
